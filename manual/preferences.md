## 介面概觀

偏好設定視窗採側邊欄＋內容區的雙欄式設計。側邊欄自上而下依序為橫幅圖片、搜尋框（⌘F）、十二個頁面圖示列表、版本編號、與官方網站連結。內容區為當前選中頁面的表單。預設進入時選中的是「一般設定」。

側邊欄列表順序如下，本文亦按此順序逐頁講解：

1. 關於唯音（`info.circle.fill`）
2. 一般設定（`wrench.and.screwdriver.fill`）
3. 選字設定（`filemenu.and.selection`）
4. 行為設定（`switch.2`）
5. 輸出設定（`text.append`）
6. 辭典設定（`text.book.closed.fill`）
7. 語彙編輯（`tablecells.badge.ellipsis`）
8. 磁帶設定（`externaldrive.fill.badge.plus`）
9. 鍵盤設定（`keyboard.macwindow`）
10. 客體設定（`square.and.pencil`）
11. 服務設定（`gearshape.fill`）
12. 開發道場（`pc`）

> 「語彙編輯」「客體設定」「服務設定」三個頁面不會在頁面內顯示搜尋欄，但仍然可被全域搜尋（⌘F 搜尋表單）找到。

每個選項後方若有灰色小字，即為該選項的官方說明文字；本文會一併引用。下文表格中「預設」欄標示的是出廠預設值。

## 頁面 11：服務設定

編輯選字窗內可由「(Shift+)符號選單鍵」呼出的服務選單內容。呼出服務選單時，原選字窗高亮選中的候選字詞會成為各功能的處理參數；若無可用服務，該熱鍵在選字窗內會改為輔助翻頁。

### 操作

| 按鈕 | 作用 |
| - | - |
| 新增服務（`+`） | 開啟輸入框貼入服務定義。 |
| 移除所選（`-`） | 移除選中條目。 |
| 上移／下移 | 調整服務在選單中的順序。 |
| 全部拷貝到剪貼簿 | 將所有服務以 `名稱\t定義` 格式拷貝。 |
| 填寫指引（`?`） | 顯示資料填寫規則。 |
| 恢復原廠設定 | 將服務清單重置為出廠預設。 |

表格兩欄：「選單條目名稱」與「定義資料值」。也可拖入文字檔以追加條目。

### 填寫規則

> 每行一筆定義，選單條目名稱與定義資料值以 Tab 字元分割。選單名稱當中的「%s」會在使用服務時被置換成傳入的參數；若參數只有一個字元，則程式會自動在選單名稱結尾用括弧追加顯示其萬國碼碼位。定義資料值的開頭是「@SEL:」（呼叫唯音內部預先定義的功能，須以西文半形冒號結尾）或「@WEB:」／「@URL:」（緊接著填網址，網址中的「%s」會被置換成參數）。

### 出廠預設服務

預設包含：Unicode Metadata、macOS Dict、Bing、DuckDuckGo、Ecosia、Google、MoeDict、Wikitonary、Unihan、Zi-Hi、HTML Ruby Zhuyin／Pinyin、Zhuyin／Pinyin Annotation、Braille 1947／2018、Baidu、BiliBili、Genshin BiliWiki、HSR BiliWiki 等共 20 筆。

## 舊版介面差異

macOS 10.13 與 10.14 使用舊版 AppKit 偏好設定介面。其頁面集合與順序與新版完全相同（十二個頁面、同樣的標題與順序），主要差異如下：

- **呈現方式**：舊版為自訂 `NSWindow`＋左側欄＋右側捲動內容區（10.13 不包捲動視窗以避免輸入法行程凍結）；新版為 SwiftUI `NavigationSplitView` 與 `Form`。舊版部分頁面（行為、選字、辭典、開發道場）以 `NSTabView` 子分頁（Ａ–Ｅ）分組，新版則攤平為表單區段。
- **圖示**：舊版在 10.13/10.14 使用傳統模板 PNG（`PrefToolbar-<名稱>`），因 SF Symbols 需 macOS 11 以上。
- **因系統版本而隱藏的選項**（舊版直接隱藏、新版則保留但停用）：
  - `通知飄窗配色外觀`：需 macOS 10.14（10.13 隱藏）。
  - `以 Caps Lock 切換…顯示通知`：需 macOS 12（10.13/10.14 隱藏）。
  - `以 Shift 切換…顯示通知`：需 macOS 10.15（10.13/10.14 隱藏）。
  - `允許使用左/右 Shift 切換英數輸入模式`：需 macOS 10.15（10.13/10.14 隱藏）。
- **以「需 macOS 10.15」標註但仍可互動的選項**（舊版渲染時自動在下方加註系統需求文字）：`不使用唯音輸入法內建的 Caps Lock 處理`、`使「Shift 鍵／JIS 英数鍵」英數模式關閉狀態與 Caps Lock 同步`。新版在 10.15 以下會將這些選項停用並把需求寫在區段註腳。
- **位置差異**：`廉恥模式`（4 月 1 日彩蛋）在舊版位於「行為設定」，新版則移至「一般設定」。
- **細節差異**：舊版「行為設定」的 BPMFVS 選項下方無 GitHub 連結；新版在英文介面下會停用「始終使用橫排來顯示工具提示視窗」。
